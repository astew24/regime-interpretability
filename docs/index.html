Interpretable Market Regime Detection
via Sparse Autoencoder Decomposition

A neural architecture study asking whether rolling multi-asset return windows can be compressed into regime embeddings and then decomposed by a sparse autoencoder into features with interpretable mappings to known market conditions.

Architecture

Two-stage encoding pipeline

The pipeline separates the compression problem from the interpretability problem. A standard autoencoder first learns a compact dense embedding from the raw return windows. A sparse autoencoder then decomposes those frozen embeddings into an overcomplete dictionary of features, enforcing sparsity so each input activates only a small subset.

The sparse autoencoder uses TopK feature selection: for each input, only the k largest activations are retained and the rest are zeroed. This forces the learned dictionary to develop specialized, non-redundant features rather than distributing regime information diffusely across all dimensions.

Key Findings

What the features learned to track

Early detection result: Sparse autoencoder features identified distribution shifts 2–4 days earlier on average than HMM and K-means baselines during held-out crisis periods — specifically COVID-19 (2020) and the 2022 rate shock.

After training, each sparse feature was correlated against external regime indicators (VIX, term spread, cross-asset correlation). Eight features showed correlations strong enough to support an interpretable label:

The low-volatility regime (feature_028) was a documented failure mode: when cross-asset correlations compressed in calm markets, multiple regime features collapsed into noisy superposition, reducing distinctiveness. Ablation studies across sparsity penalties, embedding dimensions, and window lengths confirmed this was structural, not a tuning artifact.

Code

Sparse autoencoder architecture

The sparse autoencoder applies TopK selection to enforce that each regime embedding activates at most k dictionary features. This makes individual features more specialized and easier to correlate against external indicators.

models/sparse_autoencoder.py python 
class SparseAutoencoder(nn.Module):
    """Overcomplete sparse autoencoder with TopK feature activation selection."""

    def __init__(self, input_dim: int, dict_size: int, topk: int) -> None:
        super().__init__()
        self.encoder = nn.Linear(input_dim, dict_size)
        self.decoder = nn.Linear(dict_size, input_dim, bias=False)
        self.topk = topk

    def forward(self, x: torch.Tensor):
        dense_code = self.encoder(x)

        # Retain only the k strongest activations per input
        active_indices = torch.topk(dense_code.abs(), k=self.topk, dim=1).indices
        sparse_code = torch.zeros_like(dense_code)
        sparse_values = torch.gather(dense_code, dim=1, index=active_indices)
        sparse_code.scatter_(dim=1, index=active_indices, src=sparse_values)

        reconstructed_x = self.decoder(sparse_code)
        return reconstructed_x, sparse_code, active_indices

Feature interpretability analysis — correlating sparse activations against external indicators:

analysis/interpretability.py python 
def correlate_with_indicators(
    activations: pd.DataFrame,
    indicators: pd.DataFrame,
) -> pd.DataFrame:
    """Measure linear correlation between sparse feature activations
    and external indicators (VIX, term spread, cross-asset correlations)."""

    common_dates = activations.index.intersection(indicators.index)
    a = activations.loc[common_dates]
    b = indicators.loc[common_dates]

    # Pearson correlation: each feature vs. each external indicator
    corr = pd.DataFrame(
        {ind: a.corrwith(b[ind]) for ind in b.columns},
        index=a.columns,
    )
    return corr  # shape: (dict_size, n_indicators)
